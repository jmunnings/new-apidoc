API Reference

Price API Overview

| Resource | URI | Methods | Description | | ——– | ——– | ——- | ———– | | instruments | /v1/instruments | GET | Retrieve a list of available currency pairs. | | quote | /v1/quote | GET | Retrieve live prices for specified instrument(s). | | history | /v1/history | GET | Retrieve historical rates for the instrument pair. |

Trading API Overview

| Resource | URI | Methods | Description | | ——– | ——– | ——- | ———– | | account| /v1/accounts/:account_id | GET | Contains account information for a specific account. | | account collection | /v1/accounts | POST | Contains list of accounts for a specific user. | | trade | /v1/accounts/:account_id/trades/:trade_id | GET, PUT, DELETE | Contains info of a specific trade. | | trade collection | /v1/accounts/:id/trades | GET, POST | Contains a list of trades for a specific account. Use POST to create new trades. | | order | /v1/accounts/:account_id/orders/:order_id | GET, PUT, DELETE | Contains info of a specific order. GET to retrieve info. PUT to change, DELETE to delete.| | order collection | /v1/accounts/:account_id/orders | GET, POST | Contains a list of orders for a specific account. Use POST to create new orders | | position collection | /v1/accounts/:account_id/positions | GET, DELETE | Contains a list of positions for a specific account. Use GET to retrieve. DELETE to delete existing position. | | transaction | /v1/accounts/:account_id/transactions/:trans_id | GET | Contains info of a specific transaction. | | transaction collection | /v1/accounts/:account_id/transactions | GET | Contains info of a list transactions. |

Request and Response

All requests require Content-Type: application/x-www-form-urlencoded unless specified otherwise.

All responses will be in JSON format.

Handling Errors

When an error occurs, the applicable HTTP response code is returned as well as an error message in the body in the following format:

```shell HTTP/1.1 400 Bad Request

{ “code” : [OANDA error code, may or may not be the same as the HTTP status code], “message” : [a description of the error which occurred, intended for developers], “moreInfo” : [(OPTIONAL) a link to a web page describing the error and possible causes and solutions] } ```

More on error codes

Rate Limiting

Client is allowed to have no more than 4 requests per second on average, with bursts of no more than 5 requests. Excess requests will be delayed on our server.
